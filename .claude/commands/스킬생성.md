# 스킬 생성

$ARGUMENTS 에 스킬 이름이 전달됩니다 (예: `/스킬생성 CORS-fix`).

반복 패턴을 스킬 문서로 만들고 PR을 생성합니다.

## 절차

### 1. 인자 확인
skill_name = "$ARGUMENTS"
비어있으면 "스킬 이름을 입력해주세요. 예) /스킬생성 CORS-fix" 라고 안내하고 중단.

### 2. 분석 리포트 참조
`analysis/` 디렉토리에서 가장 최근 리포트를 읽어줘.
해당 skill_name과 관련된 에러 패턴 및 발생 케이스를 찾아줘.

### 3. 스킬 문서 생성
`skills/{skill_name}.md`를 아래 형식으로 생성해줘.
관련 에러 케이스에서 패턴을 추출해서 내용을 채워줘:

```markdown
# {skill_name}

## 증상
{어떤 에러 또는 상황에서 이 스킬이 필요한지}

## 원인
{근본 원인 설명}

## 해결 방법

### 단계별 절차
1. {첫 번째 단계}
2. {두 번째 단계}
...

### 코드 예시
\`\`\`
{관련 코드 예시}
\`\`\`

## 예방법
{재발 방지 방법}

## 관련 태스크
{이 스킬이 사용된 task_id 목록}

## 작성일
{오늘 날짜}
```

### 4. git 브랜치 생성 및 커밋
```bash
git checkout -b skill/{skill_name}
git add skills/{skill_name}.md
git commit -m "skill: {skill_name} 추가"
git push origin skill/{skill_name}
```
위 명령어를 실행해줘.

### 5. PR 안내
다음 형식으로 PR 생성 안내를 출력:

```
📝 스킬 문서가 생성되었습니다!

브랜치: skill/{skill_name}
파일: skills/{skill_name}.md

PR 생성 명령어:
gh pr create --title "skill: {skill_name} 추가" --body "반복 패턴 분석으로 스킬 문서 생성"

머지 후 팀원들이 /{skill_name} 스킬을 사용할 수 있습니다.
```
