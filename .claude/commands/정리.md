# 정리

$ARGUMENTS 에 project_id가 전달됩니다 (예: `/정리 AlgoNotion`). 생략하면 전체 프로젝트를 정리합니다.

프로젝트 JSON의 완료 태스크를 동기화하고, 각 프로젝트 README를 최신 상태로 업데이트합니다.

## 절차

### 1. 본인 확인
`config.local.json`에서 member를 읽어줘.

### 2. 대상 프로젝트 결정
project_id = "$ARGUMENTS"

project_id가 비어있으면 `projects/` 폴더의 모든 .json 파일을 대상으로 해줘.
있으면 해당 프로젝트 파일(`projects/{project_id}.json`)만 대상으로 해줘.

### 3. 완료 태스크 동기화
`events.jsonl`에서 `task_done` 이벤트를 모두 읽어서 완료된 task_id 목록을 만들어줘.

각 대상 프로젝트 JSON을 읽어서:
- tasks 배열에서 완료된 task_id에 해당하는 항목에 `"status": "done"` 필드를 추가해줘.
- 이미 `"status": "done"`이면 건너뛰어줘.
- 변경이 있으면 파일을 저장해줘.

각 프로젝트에 대해 다음을 출력:
```
📦 {project_id}
  ✅ 완료 처리: {task_id 목록}
  ⏳ 미완료: {남은 task 수}개
```

### 4. 프로젝트별 README 업데이트
각 대상 프로젝트의 README.md를 찾아줘 (프로젝트 JSON의 paths 값에서 경로 확인).
README.md가 없으면 새로 생성해줘.

아래 섹션들을 최신 상태로 업데이트해줘:

#### ## 프로젝트 현황
```markdown
## 프로젝트 현황

| 항목 | 내용 |
|------|------|
| 상태 | {status} |
| 마감 | {due} |
| 담당 | {members} |
| 태그 | {tags} |
```

#### ## 태스크 현황
```markdown
## 태스크 현황

### 진행 중
| ID | 제목 | 담당 | 우선순위 | 예상 |
|----|------|------|----------|------|
| {id} | {title} | {member} | {priority} | {estimate_h}h |

### 완료
| ID | 제목 | 담당 |
|----|------|------|
| {id} | {title} | {member} |
```

README의 나머지 기존 내용(설치법, 사용법 등)은 그대로 유지해줘.
해당 섹션이 없으면 파일 맨 아래에 추가해줘.

### 5. 변경사항 요약 출력
```
🧹 정리 완료!

{project_id별 변경 요약}

📄 업데이트된 README:
  - {경로 목록}
```

### 6. git 커밋 및 push
변경된 파일들을 스테이징해서 커밋 후 push:

```bash
git add projects/ {README 경로들}
git commit -m "chore: 정리 — {날짜} ({프로젝트 목록})"
git push
```

위 명령어를 실제로 실행해줘.
