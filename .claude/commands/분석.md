# 분석 리포트 생성

팀 전체의 로그를 분석해서 반복 패턴과 병목을 찾습니다.

## 절차

### 1. 날짜 확인
오늘 날짜를 YYYY-MM-DD 형식으로 구해줘.

### 2. 전체 로그 읽기
`logs/` 디렉토리의 모든 `log-*.jsonl` 파일을 읽어줘.
각 파일에서 모든 기록을 파싱해줘.

### 3. 패턴 분석
다음 항목을 분석해줘:

**반복 에러 패턴**
- 동일한 에러 메시지가 2회 이상 나타나는 것을 찾아줘
- 어떤 멤버에서 몇 번 발생했는지 기록해줘

**병목 탐지**
- 프로젝트별 평균 duration_min을 계산해줘
- 전체 평균 대비 2배 이상 걸린 프로젝트를 표시해줘
- error_count가 높은 태스크 유형을 찾아줘

**스킬화 후보** (반복 패턴 기반)
- 같은 에러가 3회 이상: 즉시 스킬화 추천
- 같은 에러가 2회: 스킬화 고려
- error_count 평균 이상인 태스크 유형: 스킬화 검토

### 4. 리포트 파일 생성
`analysis/{날짜}-report.md`를 아래 형식으로 생성:

```markdown
# 분석 리포트 — {날짜}

## 반복 패턴
{에러별 발생 횟수 및 멤버 목록}

## 병목
{프로젝트별 평균 시간 및 이상치}

## 스킬화 후보
{우선순위별 스킬 후보 목록}

## 원시 통계
- 총 기록 건수: N
- 멤버별 작업량: {member: N건}
- 프로젝트별 총 시간: {project: N분}
```

### 5. 결과 출력
리포트 내용을 출력하고, 스킬화 후보가 있으면 `/스킬생성 {스킬명}` 명령어 사용법을 안내해줘.
