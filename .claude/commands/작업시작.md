# 작업 시작

$ARGUMENTS 에 task_id가 전달됩니다.
숫자(예: `1`)로 지정하면 오늘 명세서의 순번으로 해석합니다.
전체 ID(예: `AlgoNotion-Store`)도 그대로 사용 가능합니다.

## 절차

### 1. 본인 확인
`config.local.json`에서 member를 읽어줘.
파일이 없으면 "config.local.json이 없습니다. config.local.json.example을 복사해서 member를 설정해주세요." 안내 후 중단.

### 2. task_id 확인
task_id = "$ARGUMENTS"

task_id가 비어있으면:
"task_id를 입력해주세요. 먼저 `/오늘할일` 을 실행해 번호를 확인하세요." 안내 후 중단.

### 3. 오늘 명세서 로드
오늘 날짜(YYYY-MM-DD)를 구해줘.
`tasks/today-{날짜}-{member}.md` 파일을 읽어줘.
파일이 없으면 "명세서가 없습니다. 먼저 `/오늘할일` 을 실행해주세요." 안내 후 중단.

### 4. 숫자 ID → 전체 task_id 변환
task_id가 순수 숫자(예: "1", "2")이면:
  - 명세서에서 `### {순번}.` 패턴으로 해당 순번 항목을 찾아줘.
  - 예: "1" → `### 1. AlgoNotion-Store: ...` → task_id = "AlgoNotion-Store"
  - 해당 순번이 없으면 "번호 {task_id}에 해당하는 태스크가 없습니다." 안내 후 중단.

### 5. 태스크 조회
`projects/` 폴더의 모든 .json 파일에서 해당 task_id를 가진 태스크를 찾아줘.
없으면 "task_id '{task_id}'를 찾을 수 없습니다. projects/ 폴더를 확인해주세요." 안내 후 중단.

### 6. 이벤트 기록
`events.jsonl`에 append:
```
{"ts":"{ISO8601 현재시각}","type":"task_start","member":"{member}","task_id":"{task_id}","project":"{project}"}
```

### 7. 세션 로그 초기화
현재 작업 디렉토리에 `.session-log.jsonl` 파일을 새로 만들어줘 (기존 내용 덮어쓰기).
첫 줄에 기록:
```
{"ts":"{현재시각 HH:MM}","type":"session_start","task_id":"{task_id}","member":"{member}","project":"{project}"}
```

### 8. 출력
명세서에서 해당 task_id 섹션을 찾아 출력해줘.
"{task_id} 작업을 시작합니다. 세션 로그가 기록됩니다." 출력.
