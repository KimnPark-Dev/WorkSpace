# 작업 시작

$ARGUMENTS 에 task_id가 전달됩니다 (예: `/작업시작 AlgoNotion-Store`).
task_id 없이 실행하면 오늘 명세서만 생성합니다.

## 절차

### 1. 본인 확인
`config.local.json`에서 member를 읽어줘.
파일이 없으면 "config.local.json이 없습니다. config.local.json.example을 복사해서 member를 설정해주세요." 라고 안내하고 중단해줘.

### 2. 오늘 명세서 확인 및 자동 생성
오늘 날짜(YYYY-MM-DD)를 구해줘.
`tasks/today-{날짜}-{member}.md` 파일이 없으면 아래 절차로 자동 생성해줘:

  a. `events.jsonl`에서 오늘 해당 member의 `task_done` 이벤트 목록(task_id)을 추출해줘.
  b. `projects/` 폴더의 모든 .json 파일을 읽어서 해당 member가 담당하는 태스크만 추려줘.
     완료된 task_id는 제외해줘.
  c. 아래 형식으로 `tasks/today-{날짜}-{member}.md`를 생성해줘:

  ```
  # 오늘 할 일 — {member} / {날짜}

  ## 작업 목록

  ### {순번}. {task_id}: {title} [{priority}]
  - 프로젝트: {project id}
  - 레포: {path}
  - 배경: {background} (없으면 생략)
  - 관련 스킬: {skills} (없으면 생략)
  - 예상 시간: {estimate_h}h
  ```

  d. `events.jsonl`에 append:
  ```
  {"ts":"{ISO8601 현재시각}","type":"day_start","member":"{member}"}
  ```

### 3. task_id 처리
task_id = "$ARGUMENTS"

**task_id가 없는 경우:**
생성된 명세서 내용을 출력하고 "오늘도 파이팅!" 출력 후 종료.

**task_id가 있는 경우:**

  a. `projects/` 폴더의 모든 .json 파일에서 해당 task_id를 가진 태스크를 찾고, 해당 파일의 project id를 사용해줘.
     없으면 "task_id를 찾을 수 없습니다. projects/ 폴더를 확인해주세요." 안내 후 중단.

  b. `events.jsonl`에 append:
  ```
  {"ts":"{ISO8601 현재시각}","type":"task_start","member":"{member}","task_id":"{task_id}","project":"{project}"}
  ```

  c. 현재 작업 디렉토리에 `.session-log.jsonl` 파일을 새로 만들어줘 (기존 내용 초기화).
  첫 줄에 기록:
  ```
  {"ts":"{현재시각 HH:MM}","type":"session_start","task_id":"{task_id}","member":"{member}","project":"{project}"}
  ```

  d. `tasks/today-{날짜}-{member}.md`에서 해당 task_id 섹션을 찾아 출력해줘.

  e. "{task_id} 작업을 시작합니다. 세션 로그가 기록됩니다." 출력.
