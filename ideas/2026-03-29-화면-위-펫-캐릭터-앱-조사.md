# 화면 위 펫 캐릭터 앱 · 리소스 조사

## 핵심 요약

- 이미 시장에 다수의 경쟁작 존재 (Shimeji, Desktop Goose, WindowPet, Pets for Chrome 등)
- **크롬 익스텐션**: 브라우저 탭 내부로 제한, 진입 장벽 낮음, 바이럴 유리
- **macOS 앱**: `NSPanel` + SwiftUI로 전체 화면 오버레이 가능, 경험 풍부
- **Tauri + React**: 크로스플랫폼(Win/Mac/Linux) 데스크톱 앱 빌드에 적합한 스택
- 수익화는 캐릭터 스킨 판매, 구독형이 현실적

## 기술 구현 방법

### A. Chrome Extension

- **방식**: Content Script로 모든 페이지에 오버레이 레이어 삽입
- **렌더링**: Canvas 또는 CSS 스프라이트 애니메이션
- **제한**: 브라우저 창 안에서만 동작 (전체 데스크톱 X)
- **장점**: 설치 장벽 낮음, Chrome Web Store 배포 용이
- 참고: Shimeji는 `.crx` 패키지로 JS + 이미지 구성

### B. macOS 네이티브 앱 (Swift/SwiftUI)

- **핵심**: `NSPanel` (NSWindow 서브클래스) — `.floating` 레벨로 항상 최상위
- **투명 윈도우**: `isOpaque = false`, `backgroundColor = .clear`
- **전체 화면 오버레이**: 윈도우 레벨을 `.screenSaver`로 설정
- **마우스 인터랙션**: `NSTrackingArea`로 호버/클릭 감지
- 참고 아티클: [Translucent Overlay Window on macOS in Swift](https://gaitatzis.medium.com/create-a-translucent-overlay-window-on-macos-in-swift-67d5e000ce90), [Floating Panel in SwiftUI](https://cindori.com/developer/floating-panel)

### C. Tauri + React (크로스플랫폼 권장)

- **스택**: Tauri v2 (Rust 백엔드) + React/TypeScript 프론트엔드
- **장점**: Win/Mac/Linux 동시 지원, 번들 크기 작음
- **투명 윈도우**: `tauri.conf.json`에서 `transparent: true` 설정
- **참고 프로젝트**: [WindowPet](https://github.com/SeakMengs/WindowPet) (오픈소스, 구조 참고 가능)
- 빌드/배포 가이드: [Building a Desktop Pet with Tauri](https://crabnebula.dev/blog/building-a-desktop-pet-with-tauri/)

## 차별화 포인트 제안

1. **독자 캐릭터 IP**: 저작권 없는 오리지널 강아지/고양이 캐릭터 설계
2. **AI 반응**: LLM 연동으로 캐릭터가 현재 보는 페이지에 반응
3. **양쪽 동시 출시**: Chrome Extension (바이럴) + macOS 앱 (경험) 투트랙
4. **패트와 매트 연계**: 두 캐릭터(친구 2명)가 같은 화면에서 같이 돌아다님
